Git y GitHub: Control de Versiones

Aprende sobre el flujo de trabajo de Git y los comandos más comunes para gestionar tus proyectos de desarrollo.

Flujo de Trabajo con Git

Git es un sistema de control de versiones distribuido que te permite rastrear cambios en tus archivos y coordinar el trabajo entre múltiples desarrolladores.

Comandos Básicos de Git

  • git init - Inicializa un nuevo repositorio Git
  • git clone [url] - Clona un repositorio remoto
  • git add [archivo] - Añade archivos al área de preparación
  • git commit -m "[mensaje]" - Guarda los cambios en el repositorio local
  • git push - Sube los cambios al repositorio remoto
  • git pull - Obtiene y fusiona cambios del repositorio remoto
  • git branch - Lista las ramas del repositorio
  • git checkout [rama] - Cambia a una rama específica
  • git merge [rama] - Fusiona una rama con la actual
  • git status - Muestra el estado actual del repositorio

Flujo de Trabajo Típico

  1. Crear una nueva rama para tu feature: git checkout -b feature/nueva-funcionalidad
  2. Realizar cambios en los archivos
  3. Añadir los cambios: git add .
  4. Crear un commit: git commit -m "Descripción de los cambios"
  5. Subir los cambios: git push origin feature/nueva-funcionalidad
  6. Crear un Pull Request en GitHub
  7. Después de la revisión, fusionar los cambios en la rama principal

Mejores Prácticas

  • Hacer commits frecuentes y pequeños
  • Escribir mensajes de commit descriptivos
  • Mantener la rama principal limpia y estable
  • Revisar los cambios antes de hacer commit
  • Hacer pull antes de empezar a trabajar
